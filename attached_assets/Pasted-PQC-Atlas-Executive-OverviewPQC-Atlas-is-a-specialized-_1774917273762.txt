PQC-Atlas: Executive OverviewPQC-Atlas is a specialized static analysis and observability framework designed to solve "Cryptographic Blindness." It automates the discovery of quantum-vulnerable cryptographic algorithms (RSA, ECC) and generates a standardized Cryptographic Bill of Materials (CBOM) to map a migration path to Post-Quantum Cryptography (PQC).🛠️ What the Tool is Capable OfPQC-Atlas moves beyond simple "keyword searching" to provide deep, architectural insights:Multi-Language AST Discovery: It parses the actual structure of your code (Abstract Syntax Trees) to identify where cryptographic libraries are called, distinguishing between real implementation and dead code or comments.Standards Mapping: Every discovered primitive is cross-referenced against the Lattice Signature Database (LSDB), which maps legacy OIDs to NIST FIPS 203 (ML-KEM) and FIPS 204 (ML-DSA) standards.Compliance Gating: It acts as a CI/CD "quality gate." If a developer attempts to commit code using a non-compliant algorithm (e.g., RSA-2048), PQC-Atlas can trigger a build failure.Standardized Export: It produces a CycloneDX 1.7 compliant JSON output, allowing the tool to integrate directly with federal and enterprise GRC (Governance, Risk, and Compliance) platforms.🔍 What is Exactly Happening (The Process)When you run pqc-atlas scan --path ., the engine executes a four-stage pipeline:1. Lexical Analysis & ParsingThe tool doesn't just "read" your text files. It builds a mathematical model of your code (the AST). This allows it to understand that keySize := 2048 inside a crypto/rsa function is a high-risk vulnerability, while the same text in a README.md is harmless.2. Signature MatchingThe engine compares the identified code nodes against the LSDB.Input: CallExpr to GenerateKey with RSA.Lookup: Does this match a legacy OID? Yes.Status: Flagged as Quantum-Vulnerable.3. Risk Calculation ($QES$)The tool applies the Quantum Exposure Score formula:$$QES = (A_w \times E_f) + T_d$$It weighs the strength of the algorithm ($A_w$) against how exposed the code is ($E_f$—e.g., is it in a public API?) and adds a penalty for transitive dependencies ($T_d$—vulnerabilities found in 3rd party libraries you don't control).4. CBOM SerializationFinally, it bundles all findings into a Cryptographic Bill of Materials. This file lists:Component: The library name and version.Algorithm: The specific primitive found.Target: The NIST-approved PQC replacement (e.g., ML-KEM-768).BOM-REF: A unique identifier used to track this specific risk across the enterprise.💡 The "Elevator Pitch" for your Proposal"PQC-Atlas treats cryptography as data, not just code. By automating the discovery and inventory of cryptographic assets into a machine-readable CBOM, we provide the visibility required to meet federal PQC mandates without the friction of manual audits."
